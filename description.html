Tested webbrowsers: Internet Explorer 11(no extensive testing), Google Chrome(V34, well tested), Mozilla Firefox(V29.0.1, pretty well tested)

Installation and usage

Installation

Menuizer has been designed to be easy to use "out of the box" but still retaining a great deal of customizability so hopefully this section should be enough for you to get your menu up and running quickly.

First of all you should link the menuizer.js file in the file you intend to use Menuizer in, you should also link the menuizerStyle.css file. Depending on where you have place the downloaded js and css files the following code could look a little different to you but the only thing that should differ is the file paths.

useAjax option

The useAjax option specifies whether or not to use ajax to load the menu pages, if this is set to true you will also need to create an div into which the content will be loaded, by default menuizer tries to load into a div with the id "ajaxDiv", but you can change the name of this with the ajaxLoadDiv option.

If you do not want to use AJAX you will need to make sure that the href attribute of the <a> elements are correctly pointed to the page you want to display.

Also menuizer will try to get the page name in the following order(when using ajax):
data-url(you specify it in the html like "data-url='index'")
id(you specify it in the html like "id='index'")
href(you specify it in the html like "href='index.php'", it's important to include file-extension on this one as Menuizer does not include it automatically)
src(should probably only be used on images)

Default value: "false"
Value type: boolean(also accepts String)

redirectOnVoid option

The redirectOnVoid option specifies whether or not to redirect when "javascript:void(0)" is encountered and ajax is not used. If set to true menuizer will try to redirect to first the "data-url" and if there is no such thing it will try the id and if that too fails it will try src and if there is no src(should probably only exists on img elements) it will redirect to an error page.

Default value: true
Value type: boolean(accepts String aswell)

Load only a specific div from an page

To load only a specific element(eg. a <div>) from a page loaded using ajax(options.useAjax == true) you simply set the page-extension to ""(you could also try and set this to "fileExtension #divId" but then it will do that for all menu items) and then specify in the href the adress as follows "page.extension #element", an more concrete example could look like "index.php #myDiv".

Clickable submenu parent

To make a submenu parent clickable you first have to turn of autoCorrect as autoCorrect will assume that if an element has children that it should not be able to be pressed. Then you will need to link the href to a valid page.
A shortened down example is as follows: Turn of autoCorrect as "$("#menuId").menuize({"autoCorrect":"false"}); Link the parent item as "<li><a href="validLink.php"></a><ul> ... </ul></li>"

Anchor in the same document

If you simply want an menu which goes to a specific part of document using an anchor, you just turn of the AJAX("useAjax":"false") and the autoCorrect("autoCorrect":"false"), you can also turn of the redirectOnVoid("redirectOnVoid":false) option as this can cause weird behavior if you are using "javascript:void(0);" inside your href-attributes. And then you simply set the href parts to the anchor parts that you have created in the document.
